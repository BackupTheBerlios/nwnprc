Attack Lichs and Demi-lichs on sight

void AttackOnSight();

void AttackOnSight()

{

    //if we didnt see something (i.e heard something, stopped hearing, stopped seeing)

    //then abort

    if(GetLastPerceptionSeen()!=TRUE)

        return;

    object oTarget = GetLastPerceived();

    //if its already a hostile, abort

    if(GetIsEnemy(oTarget, OBJECT_SELF))

        return;

    //if it has a master, e.g. summon, familiar, henchman, etc

    //then abort

    if(GetIsObjectValid(GetMaster(oTarget)))

        return;

    int nAppearance = GetAppearanceType(oTarget);

    //if it matches an apearance on the list, then it is not declared hostile

    //if it doesnt match one of these appearances, then kill it

    //You can change it so that these appearances are regarded as threatening

    // by swapping the return and SetIsTemporary lines after the if statament

    //For example, if you want it to attack lichs on sight.

    if(nAppearance == APPEARANCE_TYPE_HALF_ELF

        || nAppearance == APPEARANCE_TYPE_HALF_ORC

        || nAppearance == APPEARANCE_TYPE_HALFLING

        || nAppearance == APPEARANCE_TYPE_HUMAN

        || nAppearance == APPEARANCE_TYPE_GNOME

        || nAppearance == APPEARANCE_TYPE_ELF)

    {

        return;

    }

    else

    {

        SetIsTemporaryEnemy(oTarget);

    }

}

Attach this to the OnPercieved event of townspeople, city guards, etc. Basically, if it sees someone with the appearance of a lich or demi-lich (which is probably a non-disguised PRC lich, or could be a PRC shifter), they will be enemies.
Easily extended to other appearances to cope with say, shifters in dragon shapes wandering the city centre.

 

Here is a function designed to stop people mixing elements.

How non-RP would a Elemental Savant of Fire bonded to an Cold Elemental? Or having levels in both ESFire and ESAcid?

Add the following to OnLevelUp:

if(CheckElementalCompatability(oPC) == FALSE)
{
    int nXP = GetXP(oPC);
    SetXP(oPC, (GetHitDice(oPC)*(GetHitDice(oPC)-1)*500)-1);
    DelayCommand(0.1, SetXP(oPC, nXP));
    SendMessageToPC(oPC, "You cannot mix Elemental classes");
}


and put the following function somewhere:

int CheckElementalCompatability(object oPC)
{
    int nAcidLevel = GetLevelByClass(CLASS_TYPE_ES_ACID, oPC);
    int nColdLevel = GetLevelByClass(CLASS_TYPE_ES_COLD, oPC);
    int nElecLevel = GetLevelByClass(CLASS_TYPE_ES_ELEC, oPC);
    int nFireLevel = GetLevelByClass(CLASS_TYPE_ES_FIRE, oPC);

    if(nAcidLevel>0 &&
        (nColdLevel >0 || nElecLevel>0 || nFireLevel >0))
        return FALSE;
    if(nColdLevel>0 &&
        (nAcidLevel >0 || nElecLevel>0 || nFireLevel >0))
        return FALSE;
    if(nElecLevel>0 &&
        (nColdLevel >0 || nAcidLevel>0 || nFireLevel >0))
        return FALSE;
    if(nFireLevel>0 &&
        (nColdLevel >0 || nElecLevel>0 || nAcidLevel >0))
        return FALSE;
    if(GetHasFeat(FEAT_BONDED_EARTH,oPC) &&
        (nColdLevel >0 || nElecLevel>0 || nFireLevel >0))
        return FALSE;
    if(GetHasFeat(FEAT_BONDED_AIR,oPC) &&
        (nColdLevel >0 || nAcidLevel>0 || nFireLevel >0))
        return FALSE;
    if(GetHasFeat(FEAT_BONDED_FIRE,oPC) &&
        (nColdLevel >0 || nElecLevel>0 || nAcidLevel >0))
        return FALSE;
    if(GetHasFeat(FEAT_BONDED_WATER,oPC) &&
        (nAcidLevel >0 || nElecLevel>0 || nFireLevel >0))
        return FALSE;
    return TRUE;
}

Dont forget to include both prc_feats_const and prc_class_const.

 

Making Lich's Dynamic

1) Get the HUB races v2 hakpak. http://nwvault.ign.com/Files/hakpac...803650196.shtml

2) Add that hakpak to the module. It doesnt conflict with the PRC, but it will with the CEP and many other hakpaks.

3) Change the script pnp_lich_level at about line 110 from APPEARANCE_TYPE_LICH to 506.

4) Change the script pnp_lich_alter at about line 32 from APPEARANCE_TYPE_LICH to 506.

5) Create a copy of prc_scripts.hak and remove pnp_lich_level.* and pnp_lich_alter.* from the copy.

6) Remove prc_scripts.hak from the module, and use the copy instead.

7) Levelup as a lich and enjoy being a dynamic lich!

NOTE: The only minor problem I have seen is that you have a bare skeleton head instead of the lich head, but I can live with that. If you cant, you just have to rename the head models I think. SetAppearance gives all body parts number 001.

 

NPCs and the PRC

Ok then, here is a short introduction to getting NPCs to begin to work with the new PRC classes. The PRC has added some scripts you need to hook into NPC scripts in order to have the PRC passive abilities. Simply add ExecuteScript(NAMEOFSCRIPT, OBJECT_SELF); somewhere in the AI script for each event. Most of these scripts do not do anything at the moment, but they might do in future versions.

OnBlocked prc_npc_blocked

OnCombatRoundEnd prc_npc_combat

OnConversation prc_npc_conv

OnDamaged prc_npc_damaged

OnDeath prc_npc_death

OnDisturbed prc_npc_disturb

OnHeartbeat prc_npc_hb

OnPerception prc_npc_percep

OnPhysicalAttacked prc_npc_physatt

Onrested prc_npc_rested

OnSpawns prc_npc_spawn

OnSpellCastAt prc_npc_spellat

OnUserdefined prc_npc_userdef

 

That's it! Of course, this doesnt mean that the NPC will actually use any of its new powers, but at least it is not missing out on anything. There are still some problems with NPCs

1) They don't use the active powers in any intelligent manner. They may use them as part of the Talent functions, but don't bet on it! Bioware default AI is better for this than most others. The exception that proves the rule is the epic spells. A basic wrapper AI has been implemented for many of the new epic spells. In order to use it, you dont have to do anything once the scripts are hooked as above. If you want to manually setup NPCs to have specific spells, simply add the correct feat to the character and the system will detect that they have been setup and will not give them any free epic spells. You might also want to adjust a local variable called NPC_XP, this is used to burn XP for certain epic spells that need it. It is treated exactly the same as PC XP, so that it can only go down to the minimum for the NPCs level. Once this is setup, be prepared for much tougher combats; any players will be shocked as spells such as Nailed To The Sky take out melee tanks, party members are trapped with Enslave, and the entire party is petrified with Audience Of Stone.

2) The toolset will throw up errors whenever the LevelUp or Character Wizards are used with the certain classes. We are not sure why it does it, and it only does it with certain classes. The best way around the problem is to hand make them, but your doing that anyway, right?

And plenty of other, more specific, problems.
