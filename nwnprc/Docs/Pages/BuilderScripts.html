Working with PRC scripts

back to Builders


There is an issue with prc_scripts.hak. The source scripts are not the same scripts as the compiled scripts. Which is really wierd becuase when the distribution copies are made, the scripts are compiled and the source copied straight in. As of 2.2, either the problem will be fixed, or a separate script distribution will be made.

As you may have noticed, the PRC scripts dont compile with the toolset based compiler. For some time now, the PRC has compiled its scripts with bioware provided command line compiler. This is located in the utils directoy and is named clcompile.exe I suggest that you create a shortcut to that file and change it as follows.

  1. Right click on shortcut
  2. Go to Properties
  3. In the Target box, add double quotes around the text. For example it should be something like ["C:\Games\NeverwinterNights\NWN\utils\clcompile.exe"] without [ or ].
  4. At the end, add the directory you use for NWN, then modules, then temp0, then *.nss. For example, ["C:\Games\NeverwinterNights\NWN\utils\clcompile.exe" C:\Games\NeverwinterNights\NWN\modules\temp0\*.nss]
  5. Then add the same again, but without *.nss. For example, ["C:\Games\NeverwinterNights\NWN\utils\clcompile.exe" C:\Games\NeverwinterNights\NWN\modules\temp0\*.nss C:\Games\NeverwinterNights\NWN\modules\temp0\]
  6. Then open the toolset and open any module without closing the shortcut properties.
  7. Once the module is open, close the shortcut properties by clicking OK.
  8. In the toolset, go to build->build module and make sure that "advanced controls" is checked.
  9. Then make sure that "scripts" is not checked.
  10. Then click done.
  11. Go to Tools->Options then select ScriptEditor and make sure that "Automatically compile scripts on save" is clear.
  12. Then click OK.



From now on, whenever you wish to compile scripts for a module, use the shortcut rather than the toolset. You can still use the toolset for every other purpose, but dont try and compile in there! Also, you should copy prc_scripts.hak to prc_scripts.erf and import it into every module that uses the PRC. This will ensure that all the required includes are present. It should also be done after every PRC update.

This is not a change that will break biowares toolset compiler, more like the compiler is already broken, and this is a workaround.
